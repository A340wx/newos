Setting up the source repository

If you want read only source access to the source depot, follow the following instructions and you'll be set. I'm not using CVS, so it may be a bit more different than you're used to.

Directions

  • Get a copy of the perforce client from www.perforce.com. They have it for pretty much every OS and architecture you can throw at it. They'll probably port it for you if they dont have it.
  • Set up the perforce client:
    User Name (P4USER): readonly
    Server Address (P4PORT): www.newos.org:9998
    Client Name (P4CLIENT): YourFirstnameLastname (something unique)
    If you're using the Windows GUI, you can configure this stuff with that, otherwise you'll need to set up environment variables to store this information. The above bold names are the environment variables you'll need to set before running 'p4 client'.
    For example, using bash: export P4USER="readonly"
    using csh: setenv P4USER readonly
  • If you want just the newos source and no extra stuff, only map //depot/newos/, to do this, set your client mapping line to //depot/newos/... //yourclientnamehere/newos/... instead of the default //depot/... //yourclientnamehere/...
  • Need some help with perforce? Here is a link to their documentation
  • Some basic commands: (if using command line interface)
    'p4 client': Allows you to edit your client spec
    'p4 sync': syncs you with the source
    'p4 sync -f': unconditionally grabs all of the source again, useful if you think you've gotten out of sync with the server
Have fun! If you have any questions, please contact me at geist@newos.org
